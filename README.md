# Roy McFarland

Applied AI implementation leader, operator-builder, and 4x exit executive.

I spent 15 years on the buying and implementation side of enterprise SaaS as a founder, COO, turnaround operator, and regulated-manufacturing executive. Most of the software I bought was too large, too slow, and too generalized for the work my teams actually had to do.

Now I build the lightweight, fit-for-purpose tools I always wanted as an operator.

## What I am building

I run [Brightline Labs](https://brightline.io), an applied AI software studio focused on workflow automation, internal tools, and agentic systems for real operating environments.

My current work sits at the intersection of:

- Applied AI implementation
- Enterprise workflow automation
- Production scheduling and inventory planning
- LLM agent observability, replay, and auditability
- Domain-specific tools for operators
- Agentic development workflows with independent verification

## Why this work

The useful pattern in my background is not "executive learns to code."

I wrote JavaScript before the current AI wave, published an npm package in 2015, and came back to software because agentic development tools changed what one domain expert can ship. The real edge is that I have lived inside the workflows I now automate: production planning, purchasing, compliance, inventory control, sales operations, executive reporting, and cross-functional operating cadence.

My best software ideas usually start with a sentence I heard, said, or muttered as an operator:

> There has to be a lighter way to do this.

## Operating background

Before Brightline, I led or founded four businesses to exit across four different models:

- Professional services firm sold to a national firm
- Regulated CPG company taken public on the Canadian Securities Exchange
- Biotech consultancy sold to a private equity platform
- Specialty-chemicals turnaround absorbed into a strategic rollup

I have run P&Ls, scaled manufacturing teams, managed multi-state regulated operations, led facility buildouts, implemented ERP / CRM / MRP systems, raised capital, executed acquisitions, and worked through the real-world messiness that software is supposed to help.

## How I build

I care about software that is boring in the right places and sharp where it matters.

- Domain model before interface polish
- Small systems over bloated platforms
- Typed data contracts and explicit validation
- Observability for LLM calls and agent behavior
- Human-reviewable AI workflows
- Verification before production merge
- Tools that operators can trust under pressure

My current stack includes TypeScript, Next.js, React, Node.js, Prisma, Drizzle, Postgres, MySQL/TiDB, Redis/Upstash, Resend, OpenAI APIs, Vercel, Cursor, Codex, and Manus.
